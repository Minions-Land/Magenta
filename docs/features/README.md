# Magenta Feature Roadmap

This directory contains design documents for future Magenta features.

## 🚀 Planned Features

### Self-Evolution Capabilities

- **[Agentic Updating](./agentic-updating.md)** 📝 Design Phase
  - Intelligent upstream merge instead of overwrite
  - Agent analyzes Pi changes and Magenta customizations
  - Smart conflict resolution with user confirmation
  - Auto-testing after updates
  - **Goal**: Enable self-development and self-evolution

## Status Legend

- 📝 **Design Phase** - Concept and design documented
- 🚧 **In Progress** - Currently being implemented
- ✅ **Completed** - Feature shipped and documented
- 🔮 **Future** - Ideas for consideration

## Contributing

When adding a new feature design:

1. Create a markdown file in `docs/features/`
2. Include status, priority, and related capabilities
3. Document the problem, solution, and implementation plan
4. Update this README with a brief description

## Self-Evolution Vision

Magenta aims to develop capabilities for:
- **Self-analysis**: Understanding its own codebase
- **Self-improvement**: Identifying and fixing its own issues
- **Self-adaptation**: Evolving based on upstream and user needs
- **Self-documentation**: Maintaining its own docs and knowledge base

The Agentic Updating feature is the first step toward this vision.
