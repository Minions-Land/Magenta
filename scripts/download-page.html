# Magenta 一键安装 - 用户分发方案 ## 🎯 目标 用户只需要**一条命令**或**一个链接**，就能安装和更新 Magenta，无需手动下载二进制文件。 --- ## 方案对比 | 方案 | 用户操作 | 需要服务器 | 优点 | 缺点 | |------|---------|-----------|------|------| | **方案 1：托管安装脚本** | 一条命令 | ✅ 需要 | 最简单 | 需要维护服务器 | | **方案 2：Git 仓库** | 一条命令 | ❌ 不需要 | 利用 GitHub | 用户需要 Git | | **方案 3：GitHub Release + 说明** | 点击下载链接 | ❌ 不需要 | 简单 | 需要两步操作 | --- ## 方案 1：托管安装脚本（推荐） ### 你需要做的 1. **将安装脚本上传到某个 HTTP 服务器** ```bash # 编辑 scripts/remote-install.sh，填入你的 GitHub repo 和 token vim scripts/remote-install.sh # 上传到服务器（任意 HTTP 服务器都行） # 例如：阿里云 OSS、腾讯云 COS、自己的服务器、GitHub Pages scp scripts/remote-install.sh user@your-server.com:/var/www/html/magenta-install.sh ``` 2. **给用户一条安装命令** ```bash curl -fsSL https://your-server.com/magenta-install.sh | bash ``` ### 优点 - ✅ 用户体验最好（一条命令搞定） - ✅ 脚本自动从 GitHub Releases 下载最新版本 - ✅ 不需要给用户发送二进制文件 - ✅ Token 在脚本里，用户看不到源代码 ### 安全注意 - 确保 HTTPS（避免中间人攻击） - 脚本里包含 GitHub Token（定期轮换） --- ## 方案 2：直接从 GitHub 安装（无需额外服务器） ### 实现方式 创建一个公开的 Gist 或使用 GitHub Pages 托管安装脚本。 **步骤 1：创建 GitHub Gist** 1. 访问 https://gist.github.com/ 2. 上传 `scripts/remote-install.sh`（记得填入 token） 3. 获取 raw URL，例如： ``` https://gist.githubusercontent.com/username/xxx/raw/install.sh ``` **步骤 2：给用户安装命令** ```bash curl -fsSL https://gist.githubusercontent.com/username/xxx/raw/install.sh | bash ``` ### 优点 - ✅ 不需要自己的服务器 - ✅ 一条命令安装 - ✅ GitHub 免费且稳定 ### 缺点 - ⚠️ 公开的 Gist 包含 GitHub Token（需要使用只读权限的 token） --- ## 方案 3：生成下载页面（HTML） 如果有简单的静态网页托管，可以做一个下载页面： 下载 Magenta
