# Development

Repository guidelines live in the root [`AGENTS.md`](../../../AGENTS.md).

## Setup

Node.js 22.19.0 or newer is required.

```bash
git clone git@github.com:Minions-Land/Magenta.git
cd Magenta
npm install
npm run build
```

Run the compiled TUI from the repository root:

```bash
node pi/coding-agent/dist/cli.js
```

The process keeps the shell's current working directory, so start it from the project that Magenta should edit. When testing Magenta against its own repository, run the command at the repository root.

## Checks

```bash
npm run check
npm test
```

Focused package commands:

```bash
npm run build -w @earendil-works/pi-coding-agent
npm test -w @earendil-works/pi-coding-agent
npm test -w @earendil-works/pi-coding-agent -- test/model-registry.test.ts
```

The coding-agent build expects its workspace dependencies to have been built. Use the root `npm run build` after a clean checkout or after changing `pi/ai`, `pi/agent`, `pi/tui`, or `HarnessComponentProtocol`.

## Runtime Identity

The coding-agent package keeps its published compatibility name but configures the application identity in `pi/coding-agent/package.json`:

```json
{
  "bin": { "magenta": "dist/cli.js" },
  "piConfig": {
    "name": "Magenta",
    "configDir": ".magenta",
    "binaryName": "magenta"
  }
}
```

Use constants and path helpers from `src/config.ts`; do not hard-code `pi`, `.pi`, or asset paths in new runtime code. The hidden `/debug` command writes `~/.magenta/agent/Magenta-debug.log`.

## Project Layout

```text
pi/
  ai/             LLM providers, models, auth, and transport APIs
  agent/          provider-agnostic agent loop and tool execution
  tui/            terminal UI primitives
  coding-agent/   Magenta CLI, TUI, SDK, sessions, and host adapters
HarnessComponentProtocol/
  .HCP/           protocol/runtime implementation
  _magenta/       Magenta-specific Harness adapters
```

Generated provider catalogs under `pi/ai/src/providers/*.models.ts` and generated Harness assembly files must be regenerated by their owning scripts, not edited by hand.

## Build Artifacts

Normal TypeScript builds write package output under each workspace's `dist/`. The coding-agent package copies themes, documentation, examples, bundled skills, and export assets into its distribution. `npm run build:binary -w @earendil-works/pi-coding-agent` additionally creates a Bun executable at `pi/coding-agent/dist/magenta` when Bun and all binary dependencies are available.
